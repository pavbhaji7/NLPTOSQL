# 2.5 Product Backlog (Key user stories with Desired outcomes)

**Epic 1: Develop Rule-Guided NLP Pipeline**
*   **User Story 1.1**: As a system module, I want to parse user natural language queries into distinct lemmas and parts of speech, so that core entities and verbs can be identified logically.
    *   *Outcome*: Successful parsing using NLP libraries (e.g., spaCy) to extract root words while ignoring meaningless stop words.
*   **User Story 1.2**: As a backend processor, I want to assign explicit tags (Meta, Value, Cond, AGG) to parsed tokens.
    *   *Outcome*: Database elements, data values, and mathematical operators are categorically separated.

**Epic 2: Realize Automated Schema Linking & Join Inference**
*   **User Story 2.1**: As the linker component, I want to associate identified "Meta" tags to exact Tables and Columns in the Schema.
    *   *Outcome*: Accurate mapping, even handling basic synonyms and fully qualified names.
*   **User Story 2.2**: As the query generator, I want to infer paths between multiple tables using a Breadth-First Search.
    *   *Outcome*: Multi-table queries automatically generate correct SQL JOIN statements.

**Epic 3: Interactive Visualization Frontend**
*   **User Story 3.1**: As an end user, I want to see color-coded chips representing my parsed query tokens.
    *   *Outcome*: The user visually understands how their sentence was logically dismantled.
