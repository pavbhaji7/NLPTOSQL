# 2.6 Plan of Action (Project Road Map)

The project will be executed in a structured, agile format divided across multiple sprints.

**Phase 1: Research and Architecture Design (Weeks 1-2)**
*   Conduct literature surveys and identify research gaps.
*   Draft the architectural flow bridging the frontend, API gateway, and Python backend.
*   Define the test database schema (e.g., an IMDB-style relational structure).

**Phase 2: Sprint I - Backend Pipeline Development (Weeks 3-5)**
*   Implement the Tokenizer and Lemmatization modules using `spaCy`.
*   Develop the Semantic Tagger to classify words into Meta, Value, and Condition tags.
*   Build the Schema Linker and the Breadth-First Search algorithm for relational Join inference.
*   Construct the final SQL Generator.

**Phase 3: Sprint II - Frontend Visualization and Integration (Weeks 6-8)**
*   Develop the React single-page application.
*   Build the `SchemaViewer` and `PipelineVisualizer` components.
*   Establish the API Gateway and REST endpoints using FastAPI/Python Cloud Function logic.
*   Perform end-to-end integration testing to ensure the JSON payloads correctly drive the UI components.

**Phase 4: Testing and Documentation (Weeks 9-10)**
*   Conduct edge-case testing against complex queries.
*   Compile results, evaluate performance, and finalize all documentation and reports.
