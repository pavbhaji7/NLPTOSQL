# 1.2 Problem Statement and Description

When non-technical staff need data to make decisions, they usually can't just pull it themselves. Instead, they have to submit a request to a data engineering or analytics team and wait. This back-and-forth process creates frustrating workflow bottlenecks. Analysts end up spending a massive chunk of their week writing basic, repetitive SQL queries instead of doing actual deep-dive analysis.

Furthermore, while the industry has recently shifted its focus toward deep learning and Large Language Models (LLMs) to solve this, a massive new problem has emerged: unpredictability. Modern AI models act essentially like "black boxes." They don't actually understand the mathematical structure of a database; they are just probabilistically predicting the next word in a sequence. Because of this, they are notoriously prone to hallucinations. They might confidently invent a table name that doesn't exist, misinterpret how two tables are connected, or spit out SQL code that technically looks right but fails to execute on the backend. In an enterprise environment, where running a bad query could return entirely wrong financial numbers or crash a production database, it is a risk that organizations simply will not take.
