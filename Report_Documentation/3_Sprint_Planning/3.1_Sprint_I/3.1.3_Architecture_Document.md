# 3.1.3 Architecture Document

The backend architecture developed in Sprint I relies on a decoupled, modular design. 

The pipeline initiates at the **Input Receiver**, which captures the string. It passes sequentially to the **NLP Tokenizer**, which outputs a structured array of linguistic data. This array flows into the **Semantic Tagger**, which enriches the array with our proprietary G-SQL tags. 

The enriched data is then sent to the **Schema Inference Engine**, which heavily queries the local memory representation of the relational Database Schema graph. Once the graph validates the entities and maps the necessary paths, the output is handed to the **SQL Generator**, which formats the structured data into valid SQL syntax. All these components were wrapped in a fast, lightweight FastAPI wrapper to prepare for external consumption.
