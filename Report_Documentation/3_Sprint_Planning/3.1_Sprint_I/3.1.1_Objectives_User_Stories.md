# 3.1.1 Objectives with user stories of Sprint I

**Title of Sprint I: Backend Rule-Guided NLP Pipeline Architecture**

**Objective**: The primary objective of this sprint was to lay the foundation for the transparent natural language processing engine. Instead of relying on a black-box model, the goal was to build a deterministic system capable of taking a raw string of text and algorithmically breaking it down into SQL-ready components.

**User Stories Addressed**:
1. *Tokenization*: As a backend module, I need to strip punctuation, handle stop words, and extract the root lemma of every word so the system can understand variations of verbs and nouns.
2. *Tagging*: As the pipeline, I need to tag words that refer to database architecture (Meta) versus actual data points (Values).
3. *Schema Linking*: As the inference engine, I need to ensure that every Meta tag maps strictly to a known entity in the `DatabaseSchema.json` file.
