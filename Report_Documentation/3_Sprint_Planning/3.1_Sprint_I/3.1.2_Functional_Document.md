# 3.1.2 Functional Document

During this sprint, the core functionality of the backend was realized through several Python modules.

The `nlp.py` module was built to utilize the `spaCy` en_core_web_sm model. When a string like "Find movies released after 2000" is submitted, this function strips the text and isolates the intent. 

The `tagger.py` module evaluates these tokens. "Movies" is tagged as a `META` entity, "released after" implies a `COND` (condition: >), and "2000" is classified as a `VALUE`. 

Most crucially, the `linker.py` function was established. This function loads the database schema and performs validation. It ensures that the `META` tags actually exist as columns or tables. Furthermore, if a query references two distinct tables, it executes a Graph-Based Breadth-First Search (BFS) to find the shortest relational path between primary and foreign keys, forming the basis for SQL JOINs.
