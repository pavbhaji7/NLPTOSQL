# 3.1.5 Sprint Retrospective

Looking back on Sprint I, the team successfully established the core processing logic of the NL2SQL engine. 

**What went well:** The modular breakdown of the NLP tasks (tokenization, tagging, and linking) made debugging highly efficient. The integration of `spaCy` provided excellent, lightweight grammatical parsing without the heavy overhead of massive neural networks. The BFS algorithm for join inference proved exceptionally robust.

**What could be improved:** Handling deeply ambiguous synonyms remained a slight bottleneck. In future iterations, we recognized the need to implement either a fuzzy-matching library or a secondary lightweight LLM fallback specifically for mapping highly contextual synonyms to schema columns, ensuring the deterministic pipeline doesn't break on rare vocabulary.
