# 3.2.5 Sprint Retrospective

**What went well:** The decision to avoid heavy CSS frameworks and use vanilla CSS allowed for a highly customized, glassmorphism-inspired aesthetic that feels premium. The state management between the query input and the visualization components was smooth, and the API payload design from Sprint I proved perfectly suited for React's rendering lifecycle.

**What could be improved:** While the visualizer is excellent, the UI currently doesn't allow the user to "correct" a chip if the system misidentified a word. Future enhancements should include making the visual chips interactive, allowing users to manually override a tag (e.g., changing a Meta tag to a Value tag) and re-running the inference, creating a truly collaborative human-in-the-loop system.
