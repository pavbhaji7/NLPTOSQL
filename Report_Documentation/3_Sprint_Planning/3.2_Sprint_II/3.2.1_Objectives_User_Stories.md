# 3.2.1 Objectives with user stories of Sprint II

**Title of Sprint II: Interactive Visualization and API Integration**

**Objective**: The primary objective of Sprint II was to build the user interface and connect it securely to the Python backend developed in Sprint I. Recognizing that user trust is the largest hurdle in AI adoption, the goal was to make the backend's logic entirely transparent through a web-based dashboard.

**User Stories Addressed**:
1. *Query Interface*: As a user, I want a clean, simple search bar to type my English questions.
2. *Schema Transparency*: As a user, I want to see a visual representation of the database tables so I know what information is available.
3. *Pipeline Visualization*: As an end-user, I want to see exactly how the computer broke down my sentence into database logic, represented by color-coded chips, so I can trust the resulting SQL query.
