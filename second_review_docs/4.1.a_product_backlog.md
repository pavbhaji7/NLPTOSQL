# 4.1.a Product Backlog (MS Planner Format)

*This backlog formats research objectives as Epics. You can easily copy these Epics into MS Planner "Buckets" or "Labels", and add the User Stories as individual "Tasks" with the Acceptance Criteria placed in the task description or checklist.*

---

## Epic 1: Develop Rule-Guided NLP Pipeline
**Objective:** Implement a transparent natural language processing pipeline that avoids black-box LLM hallucinations.

### User Story 1.1: Tokenization and Lemmatization
**As a** system module,
**I want to** parse user natural language queries into distinct lemmas and parts of speech,
**So that** core entities and verbs can be identified logically.

**Acceptance Criteria:**
- [ ] Incorporate `spaCy` (en_core_web_sm) to parse input strings.
- [ ] Output a structured list of tokens containing text, lemmas, and POS tags.
- [ ] Ignore stop words that do not contribute to SQL generation (like "the", "a").

### User Story 1.2: Semantic Tagging (G-SQL Tags)
**As a** backend processor,
**I want to** assign explicit tags (Meta, Value, Cond, AGG) to parsed tokens,
**So that** database elements, data values, and operators are categorically separated.

**Acceptance Criteria:**
- [ ] "Meta" tags accurately identify database tables and columns.
- [ ] "Value" tags correctly capture requested data (e.g., "Avatar").
- [ ] "Cond" tags capture mathematical operators (>, <, =, after, before).

---

## Epic 2: Realize Automated Schema Linking & Join Inference
**Objective:** Dynamically map tagged natural language data to the predefined database schema and construct necessary relational joins.

### User Story 2.1: Schema Mapping
**As the** linker component,
**I want to** associate identified "Meta" tags to exact Tables and Columns in the `DatabaseSchema`,
**So that** the system knows exactly which columns to select and filter.

**Acceptance Criteria:**
- [ ] Successfully map synonym tokens (e.g., "film" -> "Movie" table).
- [ ] Handle ambiguous columns by reverting to fully qualified names (Table.Column).

### User Story 2.2: Graph-Based Breadth-First Join Inference
**As the** query generator,
**I want to** infer paths between multiple tables using a Breadth-First Search over primary/foreign key relationships,
**So that** multi-table queries (e.g., "Find directors of Titanic") automatically generate the correct SQL `JOIN` statements.

**Acceptance Criteria:**
- [ ] Construct an adjacency graph based on the `DatabaseSchema` foreign keys.
- [ ] Successfully path-find across at least 3 connected tables.
- [ ] Fail gracefully or output a warning if an un-joinable table is queried.

## Epic 3: Develop Interactive Visualization Frontend
**Objective:** Build a user interface that makes the transparent NLP pipeline visible and educational for end users.

### User Story 3.1: Pipeline Visualization Dashboard
**As an** end user,
**I want to** see color-coded chips representing my parsed query tokens,
**So that** I understand how my natural language sentence was logically dismantled.

**Acceptance Criteria:**
- [ ] Render a React interface that visually differentiates Meta, Value, and Cond tags with distinct styles/colors.
- [ ] Fetch translation states from the FastAPI backend asynchronously without page reload.

### User Story 3.2: Database Schema Viewer
**As an** end user,
**I want to** view a representation of the available tables and columns on the screen,
**So that** I know what kind of questions I am allowed to ask the system.

**Acceptance Criteria:**
- [ ] Create a static or dynamic UI panel indicating tables like "Movie" and "Actor" alongside their data types.
