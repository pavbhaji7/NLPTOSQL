# 4.1.c Functional Document

*This document outlines the core functional requirements and modules of the G-SQL translation system.*

## 1. System Overview
G-SQL provides users with the ability to type intuitive natural language questions, which the system parsing engine deterministically maps to an underlying relational schema (e.g., IMDB structure) and translates into structurally sound SQL statements. The application includes both a robust Python backend and a React visualization frontend.

---

## 2. Functional Modules

### 2.1 Natural Language Processing (NLP) Input 
**Description:** Captures the raw user sentence and decomposes it for logical analysis.
*   **Functionality:** 
    *   Takes unstructured English sentences.
    *   Uses `spaCy` to run linguistic annotations (Tokenization, Lemmatization, POS Tagging).
    *   Strips away irrelevant "stop words" to focus only on structural nouns, targets, and conditionals.

### 2.2 Semantic Tag Classification
**Description:** Assigns explicit schema-aware categories to language tokens based on established dictionary rules.
*   **Functionality:**
    *   Matches identified entities against a dictionary mapping.
    *   **Meta Tagging:** Determines if a token corresponds to a Schema `Table` (e.g., "Movie") or a `Column` (e.g., "release_year").
    *   **Value Tagging:** Extracts literal query inputs (e.g., looking for "Titanic" or "Male" actors).
    *   **Condition Tagging:** Interprets comparative terms (`>`,`<`,`=`,`after`,`before`).
    *   **Aggregation Tagging:** Identifies operations like `sum`, `count`, and `average`.

### 2.3 Relational Schema Linking and Join Inference
**Description:** Maps semantic tags directly into the structured SQL environment and builds required Foreign Key boundaries.
*   **Functionality:**
    *   Resolves synonyms directly to base Table paths (e.g., mapping "film" to `Movie.title`).
    *   Identifies cross-table queries. When columns from disconnected tables are present simultaneously, the engine performs a Breadth-First Search (BFS) through an abstract representation of Primary and Foreign Keys connecting tables.
    *   Returns necessary internal SQL `JOIN ON` clause conditions dynamically.

### 2.4 Automatic SQL Generation
**Description:** Reconstructs the mapped objects into formal SQL syntax.
*   **Functionality:**
    *   Combines the gathered `SELECT` vectors, `FROM` sets, derived `WHERE` clauses, and dynamic `JOIN` statements.
    *   Outputs a syntactically correct SQL string ready for execution against a relational database.

### 2.5 React Pipeline Visualization Subsystem
**Description:** A purely educational and verification tool for users to audit the process.
*   **Functionality:**
    *   Provides a schema viewer panel detailing available tables/foreign keys.
    *   Takes the JSON payload from the backend containing intermediate step data.
    *   Outputs "Tag Chips" (Visual elements highlighting exactly which words were mapped to Meta, Value, or Condition).
    *   Displays the final compiled SQL.
