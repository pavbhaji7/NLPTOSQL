**Title:** G-SQL: A Schema-Aware and Rule-Guided Framework for Robust Natural Language to SQL Translation

**Abstract**—
This research introduces G-SQL, a schema-aware translation framework that utilizes deterministic rule-guided inference for robust Natural Language to SQL mapping. The architecture integrates semantic token tagging with a graph-based join resolution engine, employing breadth-first search to autonomously infer multi-relational paths across complex database schemas. G-SQL addresses the transparency limitations of stochastic language models by providing a verifiable, step-by-step pipeline for query synthesis. The implemented framework demonstrates zero hallucination regarding schema layout and excels at complex multi-table JOIN inferences without explicit neural prompting.

**I. INTRODUCTION**
The translation of Natural Language Queries (NLQ) directly into formal Structured Query Language (SQL) has become a primary bottleneck for data accessibility in non-technical userbases. While modern advancements in Large Language Models (LLMs) have produced sophisticated "black-box" generative solutions, these models inherently suffer from non-deterministic behavior and frequent schema hallucinations. 

To bridge this gap, this paper presents **G-SQL**, a deterministic, rule-guided pipeline engineered to interpret unstructured language and output functionally verifiable databases operations. By separating the NLP translation task into discrete elements—Tokenization, Semantic Tagging, Schema Linking, and Structured Generation—the proposed framework restores explainability and trust to the translation environment.

**II. LITERATURE REVIEW**
Early approaches to Semantic Parsing for SQL generation relied heavily on rigid template matching, which constrained the variety of permissible linguistic structures. Recent progress in Seq-to-Seq Deep Learning and Transformer models (e.g., GPT-based Text-to-SQL) has allowed for incredible syntactic flexibility. However, these systems struggle with explicit schema fidelity; they regularly infer relationships or columns that do not exist within the targeted relational database. 

Rule-based bridging logic has been proven to enforce strict schema adherence, indicating a need for a hybrid or fully deterministic algorithmic pipeline that marries modern NLP contextualization with algorithmic graph traversal.

**III. PROPOSED METHODOLOGY**
The G-SQL mechanism avoids prompt engineering in favor of a formalized four-stage processing architecture:

*A. Natural Language Processing (NLP) Input*
Raw queries are passed through linguistic parsers (implemented via the `spaCy` library) to establish morphological characteristics like Lemmatization, Base Word, and Part of Speech (noun, verb, proper noun). Structural "stop words" are discarded prior to semantic mapping.

*B. Semantic Tag Classification*
Tokens are algorithmically assigned one of four logical tags:
1. **Meta:** Corresponding to distinct Database Tables (e.g., "Movie") or Columns (e.g., "Budget").
2. **Value:** Explicit search parameters or literal strings (e.g., "Titanic").
3. **Cond:** Comparative heuristics (`>`, `<`, `=`).
4. **AGG:** Aggregation functionality (`COUNT`, `SUM`).

*C. Relational Schema Linking*
An abstract mapping function explicitly binds tagged elements to a declared programmatic schema structure. Noun combinations are reduced to absolute SQL identities (e.g., matching "film" to `Table: Movie`). Values are mapped to the preceding `Column` definition.

*D. Graph-Based Join Inference*
Cross-table queries necessitate robust pathfinding. The structural metadata connecting Primary and Foreign Keys acts as an adjacency list. The engine performs an automated **Breadth-First Search (BFS)** graph traversal to define requisite SQL `JOIN ON` statements dynamically.

**IV. IMPLEMENTATION**
The system was implemented using Python for the core Engine (FastAPI routing) and a lightweight modern React interface (Vite) for presentation. The UI features a decoupled Schema Viewer and a "Pipeline Visualizer." The visualizer renders intermediate token JSON payloads into colored UI components, actively demonstrating the classification logic to users prior to displaying the final SQL syntax.

**V. RESULT ANALYSIS**
Analysis across a 5-table schema sample (Movies, Actors, Directors) demonstrates high precision mapping. G-SQL successfully eliminates non-existent schema references (zero-hallucinations) common in LLM outputs. Furthermore, the BFS algorithm efficiently resolves multi-table pathfinding, successfully linking tables separated by intermediary associative entities (e.g., linking "Actor" to "Movie" via the "Cast" junction).

**VI. CONCLUSION**
G-SQL stands as an effective framework for resolving NL-to-SQL tasks where deterministic reliability outweighs purely generative flexibility. By mapping tokenized semantics to explicit schema graphs and resolving JOINs via BFS heuristics, the system provides an interpretable alternative to stochastic models. Future iterations will focus on persisting conversation memory to handle sequential NLQ refinement.
